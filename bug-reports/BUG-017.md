# BUG-017 — All `<button>` elements default to `type="submit"`

| Field        | Value                                              |
| ------------ | -------------------------------------------------- |
| **ID**       | BUG-017                                            |
| **Status**   | Open                                               |
| **Severity** | Low                                                |
| **Priority** | P3                                                 |
| **Component**| HTML hygiene                                       |

## Summary

None of the 24 calculator buttons declare `type="button"`. The HTML
default for `<button>` elements is `type="submit"`, which would submit
the parent `<form>` if the calculator is ever embedded in one.
Today there is no parent form — but defensive hygiene matters,
especially when a feature ships with the expectation of being embedded.

## Steps to reproduce (synthetic)

1. Wrap the calculator's `.calculator` div in a `<form>`.
2. Click any button.
3. Observe the page reload (form submission).

## Expected behaviour

All buttons explicitly declare `type="button"`.

## Actual behaviour

All 24 buttons rely on the default, which is `type="submit"`.

## Evidence

| Type | Path |
|--|--|
| Failing test | `tests/ux/test_ux_and_accessibility.py::TestButtonAttributes::test_all_buttons_have_explicit_type_button` |

## Suggested fix

Add `type="button"` to every button. A find-and-replace fixes all 24
in one commit.

## Severity justification

**Low.** Latent bug today; only manifests if the calculator is later
embedded. Cheap to prevent now; potentially confusing to debug later.
