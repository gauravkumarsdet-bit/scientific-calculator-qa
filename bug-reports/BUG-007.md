# BUG-007 — `sin(x)` ignores its input and always returns `1`

| Field        | Value                                              |
| ------------ | -------------------------------------------------- |
| **ID**       | BUG-007                                            |
| **Status**   | Open                                               |
| **Severity** | Critical                                           |
| **Priority** | P0                                                 |
| **Component**| `func` — sin handler                               |

## Summary

The `sin` button's handler does not call `Math.sin` at all. Instead it
assigns the result of a constant bitwise-XOR expression — which always
evaluates to `1` — to the display. The function is therefore broken
for every input value.

## Steps to reproduce

1. Open the calculator.
2. Press `0` (so the display shows `0`).
3. Press `sin`.

## Expected behaviour

`sin(0) = 0`. Display shows `0`.

## Actual behaviour

Display shows `1`. Repeats for every input — `sin(1)`, `sin(π)`,
`sin(0.5)` all display `1`.

(`sin(π/2)` *coincidentally* matches the bug's output of `1`, which
makes a single-value smoke test misleading; the fix needs at least one
non-`π/2` test value.)

## Evidence

| Type | Path |
|--|--|
| Failing tests | `tests/scientific/test_scientific_functions.py::TestSinIsConstant::*`, and `TestTrigFunctions::*[sin_zero]` / `[sin_pi]` |

## Root cause hypothesis

In the obfuscated `func()` body:

```js
if (type === "sin") display.value = 434563 ^ 434562;
```

`434563 XOR 434562 === 1` (the two integers differ only in the lowest
bit). The input value is read into `_0xc2af5d` but never used in the
sin branch.

## Suggested fix

```diff
- if (type === "sin") display.value = 434563 ^ 434562;
+ if (type === "sin") display.value = Math.sin(_0xc2af5d);
```

(All other trigonometric branches are correctly wired to `Math.cos`,
`Math.tan`, etc., so the surrounding code is a working template.)

## Severity justification

**Critical.** The function is completely non-functional, but the
output (`1`) is plausible and produced silently — users may
unknowingly rely on it. Trivial fix; should be P0.
