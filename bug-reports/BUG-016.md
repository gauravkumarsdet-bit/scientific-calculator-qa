# BUG-016 — No viewport `<meta>`; layout is broken on mobile

| Field        | Value                                              |
| ------------ | -------------------------------------------------- |
| **ID**       | BUG-016                                            |
| **Status**   | Open                                               |
| **Severity** | Medium                                             |
| **Priority** | P2                                                 |
| **Component**| HTML head — responsive design                      |

## Summary

The page lacks `<meta name="viewport">`. On mobile devices the page is
rendered at the default 980px width and zoomed out, making the
calculator unusable without manual zoom. The fixed-pixel
`grid-template-columns: repeat(4, 70px)` compounds the issue.

## Steps to reproduce

1. Open the calculator on a mobile device, or in a desktop browser
   with the device-emulation toolbar set to "iPhone 14".

## Expected behaviour

The calculator scales to fit the viewport.

## Actual behaviour

The calculator renders at desktop size, zoomed-out, with tiny buttons.

## Evidence

| Type | Path |
|--|--|
| Failing test | `tests/ux/test_ux_and_accessibility.py::TestPageAccessibility::test_viewport_meta_is_present_for_mobile` |

## Suggested fix

```diff
  <head>
+   <meta name="viewport" content="width=device-width, initial-scale=1">
    <meta charset="UTF-8">
```

(Optionally also relax the fixed grid to `repeat(4, 1fr)` for
better mobile layout, but the meta tag alone fixes the unusability.)

## Severity justification

**Medium.** Mobile is now a majority share of web traffic; an
unusable mobile experience for a web-deployed calculator is a real
defect, not a nice-to-have.
