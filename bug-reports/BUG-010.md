# BUG-010 — Malformed parentheses do not reach the existing `Error` pathway

| Field        | Value                                              |
| ------------ | -------------------------------------------------- |
| **ID**       | BUG-010                                            |
| **Status**   | Open                                               |
| **Severity** | Medium                                             |
| **Priority** | P2                                                 |
| **Component**| Parser — error reporting                           |

## Summary

Several malformed-parenthesis inputs that should clearly be `Error`
either leak `NaN` or are silently truncated to a plausible number.
The application already has a working `Error` pathway (used for
`(` alone and for trailing operators); these inputs should reach it.

## Steps to reproduce

1. Open the calculator.
2. Enter each of the inputs in the table below and press `=`.

## Expected vs. actual

| Input | Expected | Actual |
|---|---|---|
| `)`     | `Error` | `NaN` |
| `()`    | `Error` | `NaN` |
| `(2+5`  | `Error` | `7`    |
| `2+5)`  | `Error` | `7`    |

## Evidence

| Type | Path |
|--|--|
| Failing tests | `tests/edge_cases/test_edge_cases.py::TestMalformedParenthesesShouldYieldError::*` |

## Root cause hypothesis

* `)` and `()` reach the parser with no factor to start; the LHS is
  `undefined`, arithmetic on `undefined` yields `NaN`, which leaks out.
* `(2+5` — the parser permissively accepts a missing closing `)`
  (the `if (peek === ')') consume()` branch is non-mandatory).
* `2+5)` — the trailing `)` is left unconsumed and not flagged.

## Suggested fix

* In `factor()`: if `peek() === '('` then a closing `)` must follow
  the inner expression, throw otherwise.
* In `calculate()`'s evaluator entry point: assert the token cursor
  has consumed *all* tokens; throw if not.
* Catch is already in `calculate()` — once the throws are added, all
  four cases land on `Error` automatically.

## Severity justification

**Medium.** Same general theme as BUG-005 / BUG-006 — inconsistent
error reporting. Not data-corrupting on its own, but contributes to
a broader pattern of "the calculator silently accepts garbage".
