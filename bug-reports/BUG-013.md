# BUG-013 — Display has no `aria-label`

| Field        | Value                                              |
| ------------ | -------------------------------------------------- |
| **ID**       | BUG-013                                            |
| **Status**   | Open                                               |
| **Severity** | Medium                                             |
| **Priority** | P2                                                 |
| **Component**| Accessibility — display element                    |

## Summary

The display `<input>` has no associated `<label>` and no `aria-label`.
Screen readers therefore announce only "edit" or "text input" — users
have no way to know what the field represents.

## Steps to reproduce

1. Open the calculator with a screen reader.
2. Move focus to the display.
3. Listen to what is announced.

## Expected behaviour

The screen reader announces something like
*"Calculator display, edit, read only"*.

## Actual behaviour

The screen reader announces only *"edit"* (or skips it entirely if
`disabled` — see BUG-012).

## Evidence

| Type | Path |
|--|--|
| Failing test | `tests/ux/test_ux_and_accessibility.py::TestDisplayAccessibility::test_display_should_have_aria_label` |

## Suggested fix

```diff
- <input type="text" id="display" disabled>
+ <input type="text" id="display" readonly aria-label="Calculator display">
```

## Severity justification

**Medium.** Standard WCAG 1.3.1 / 4.1.2 issue. Trivial fix.
