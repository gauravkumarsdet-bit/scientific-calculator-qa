# BUG-014 — Display is not declared as an `aria-live` region

| Field        | Value                                              |
| ------------ | -------------------------------------------------- |
| **ID**       | BUG-014                                            |
| **Status**   | Open                                               |
| **Severity** | Medium                                             |
| **Priority** | P2                                                 |
| **Component**| Accessibility — display element                    |

## Summary

When `=` is pressed, the display value updates without any focus
change. Without `aria-live`, screen-reader users will never hear the
result — which is the entire point of using a calculator.

## Steps to reproduce

1. Open the calculator with a screen reader.
2. Enter `7 + 2 =`.
3. Listen for the result.

## Expected behaviour

The screen reader announces the new display value
("nine") because the result region is declared live.

## Actual behaviour

No announcement. The user has no idea the calculation completed.

## Evidence

| Type | Path |
|--|--|
| Failing test | `tests/ux/test_ux_and_accessibility.py::TestDisplayAccessibility::test_display_should_be_an_aria_live_region` |

## Suggested fix

```diff
- <input type="text" id="display" readonly aria-label="Calculator display">
+ <input type="text" id="display" readonly
+        aria-label="Calculator display" aria-live="polite">
```

## Severity justification

**Medium.** Combined with BUG-012 / BUG-013, this is the difference
between "screen reader users *technically can* use the calculator" and
"screen reader users *can* use the calculator". WCAG 4.1.3 (Status
Messages).
