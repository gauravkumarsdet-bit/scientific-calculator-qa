# BUG-008 — `sqrt`/`log` domain errors leak `NaN` / `-Infinity` to the UI

| Field        | Value                                              |
| ------------ | -------------------------------------------------- |
| **ID**       | BUG-008                                            |
| **Status**   | Open                                               |
| **Severity** | High                                               |
| **Priority** | P1                                                 |
| **Component**| `func` — sqrt / log error handling                 |

## Summary

When a scientific function is called with an out-of-domain input
(`sqrt(-x)`, `log(0)`, `log(-x)`), the raw JavaScript `NaN` or
`-Infinity` value is coerced to a string and shown to the user. The
application already has a clean `Error` pathway used for empty input;
that pathway should also catch domain errors.

## Steps to reproduce — case A (`sqrt(-4)`)

1. Open the calculator.
2. (Negative numbers cannot be entered through the broken UI; in tests
   we set the display to `-4` directly via JavaScript to isolate the
   function bug from BUG-001.)
3. With display showing `-4`, press `√`.

## Expected behaviour

Display shows `Error`.

## Actual behaviour

| Input | Function | Got | Expected |
|---|---|---|---|
| `-4` | `√` | `NaN` | `Error` |
| `0`  | `log` | `-Infinity` | `Error` |
| `-1` | `log` | `NaN` | `Error` |

## Evidence

| Type | Path |
|--|--|
| Failing tests | `tests/scientific/test_scientific_functions.py::TestDomainErrorsAreReported::*` |

## Root cause hypothesis

In `func()`:

```js
let v = parseFloat(display.value);
if (isNaN(v)) { display.value = 'Error'; return; }
…
if (type === 'sqrt') display.value = Math.sqrt(v);
if (type === 'log')  display.value = Math.log10(v);
```

Only the *parse failure* triggers `Error`. `Math.sqrt(-4)` returns
`NaN` (a valid numeric result, never re-checked), and is assigned
directly. Same for `Math.log10(0)` returning `-Infinity`.

## Suggested fix

After computing the result, normalise non-finite values:

```js
const r = Math[fn](v);
display.value = Number.isFinite(r) ? r : 'Error';
```

(This single fix also resolves BUG-005, BUG-006 and BUG-011.)

## Severity justification

**High.** Domain errors are common (any user calculating logarithms
will eventually hit `log(0)` or `log(-x)`), and the leaked JS literal
`-Infinity` looks broken even when it's mathematically defensible.
