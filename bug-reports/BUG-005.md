# BUG-005 — Pressing `=` with empty display leaks `'undefined'` to the UI

| Field        | Value                                              |
| ------------ | -------------------------------------------------- |
| **ID**       | BUG-005                                            |
| **Status**   | Open                                               |
| **Severity** | Medium                                             |
| **Priority** | P2                                                 |
| **Component**| `calculate` / `evaluateExpression`                 |

## Summary

Pressing `=` with nothing entered displays the literal string
`undefined` rather than staying empty or surfacing `Error`. This leaks
internal JavaScript into the UI and looks unprofessional.

## Steps to reproduce

1. Open the calculator.
2. Without entering anything, press `=`.

## Expected behaviour

Display remains empty, or shows `Error` (matching the existing error
pathway used for trailing-operator inputs).

## Actual behaviour

Display shows `undefined`.

## Evidence

| Type | Path |
|--|--|
| Failing test | `tests/functional/test_digits_and_decimal.py::TestClearAndErrors::test_pressing_equals_with_empty_display_must_not_show_undefined` |

## Root cause hypothesis

`evaluateExpression('')` returns `undefined` (the function explicitly
short-circuits when the token list is empty). `calculate` then assigns
the return value to `display.value`, which coerces `undefined` to the
string `"undefined"`.

## Suggested fix

In `calculate`:

```js
const r = evaluateExpression(display.value);
if (r === undefined || Number.isNaN(r)) {
  display.value = '';   // or 'Error'
  return;
}
display.value = r;
```

## Severity justification

**Medium.** Cosmetic leak, but a very visible one — instantly breaks
user trust. Easy to discover and trivial to fix.
