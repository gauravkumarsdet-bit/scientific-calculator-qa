# BUG-001 — Minus button performs division instead of subtraction

| ------------ | -------------------------------------------------- |

| **Reported** | 2026-05-07                                         |
| **Affects**  | `https://rbihubcodechallenge.github.io/calculator/`|

## Summary

The button visibly labelled `−` (Unicode U+2212 MINUS SIGN) is wired to
append `'/'` to the display, so every subtraction the user attempts
becomes a division. Combined with BUG-003 (division operand-swap), the
button silently produces wildly wrong but plausible-looking results.

## Environment

| | |
|--|--|
| URL | <https://rbihubcodechallenge.github.io/calculator/index.html> |
| Browser | Chromium (latest, headless)                                |
| OS | macOS 14.6                                                       |

## Steps to reproduce

1. Open the calculator.
2. Press `9`, then `−`, then `4`, then `=`.
3. Observe the display.

## Expected behaviour

`9 − 4 = 5` — display shows `5`.

## Actual behaviour

Display shows `0.4444444444444444` — the result of `4 / 9`.

(The minus button appended `/` instead of `-`, turning the input into
`9/4`. The operand-swap defect BUG-003 then evaluated that as `4/9`.)

## Evidence

## Root cause hypothesis

In `index.html` the minus row contains:

```html
<button onclick="append('/')">−</button>
```

The handler should be `append('-')`. Almost certainly a copy-paste
slip from the divide row immediately above it.

## Suggested fix

```diff
- <button onclick="append('/')">−</button>
+ <button onclick="append('-')">−</button>
```

## Severity justification

**Critical.** Subtraction is core calculator functionality and the
defect produces a *plausible* wrong answer rather than an obvious error,
so users will trust and act on the bad number. There is no workaround
that does not require knowing about the bug.

## Notes / related bugs

* BUG-003 (division operand-swap) compounds this defect — the actually
  observed value is `b / a` rather than `a / b`, making the failure
  even harder to debug from the user's side.
