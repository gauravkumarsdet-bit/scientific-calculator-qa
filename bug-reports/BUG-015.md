# BUG-015 — No semantic landmark or heading on the page

| Field        | Value                                              |
| ------------ | -------------------------------------------------- |
| **ID**       | BUG-015                                            |
| **Status**   | Open                                               |
| **Severity** | Low                                                |
| **Priority** | P3                                                 |
| **Component**| Accessibility — document structure                 |

## Summary

The page has neither a `<main>` landmark nor any `<h1>` / `<h2>`
heading. Screen-reader users navigate by landmarks and headings; the
only signal of what the page is, is the `<title>` element.

## Steps to reproduce

1. Open the calculator with a screen reader.
2. Try landmark navigation (e.g. VoiceOver Rotor → Landmarks).

## Expected behaviour

A "Main" landmark and at least one `<h1>` are discoverable.

## Actual behaviour

No landmarks; no headings.

## Evidence

| Type | Path |
|--|--|
| Failing test | `tests/ux/test_ux_and_accessibility.py::TestPageAccessibility::test_page_has_main_landmark_or_heading` |

## Suggested fix

```diff
- <body>
+ <body>
+   <h1 class="visually-hidden">Scientific Calculator</h1>
+   <main>
    <div class="calculator">…</div>
+   </main>
- </body>
+ </body>
```

## Severity justification

**Low.** The calculator is a single-purpose page; landmark navigation
is less critical than on content-heavy pages. Still trivially fixable
and counts toward the WCAG 1.3.1 / 2.4.1 set.
